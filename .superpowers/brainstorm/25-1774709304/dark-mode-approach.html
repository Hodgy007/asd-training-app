How should dark mode be controlled?

Pick the approach that works best for your users

A

Manual toggle in the app

A sun/moon button in the topbar. Users switch whenever they want — default follows their system setting on first visit. Persistent (remembered via localStorage).

Pros

  • Full user control
  • Familiar UX pattern
  • Works independently of OS setting

Cons

  • Adds a button to the topbar
  • Requires a ThemeProvider component
B

Follow system preference only

Uses the OS dark/light setting automatically via prefers-color-scheme CSS media query. No toggle needed — it just works.

Pros

  • Zero UI complexity
  • Respects OS accessibility setting
  • Less code

Cons

  • User can't override without changing OS
  • Less control for users on shared devices
C

Toggle in Settings page

No topbar button — instead a "Theme" dropdown (Light / Dark / System) lives in the Settings page the user already has access to.

Pros

  • Keeps topbar clean
  • Three-way control (Light / Dark / System)

Cons

  • Less discoverable — buried in settings
  • Harder to switch quickly

👉 My recommendation is A — a topbar toggle is the most accessible and discoverable pattern for users who may have visual impairments.
